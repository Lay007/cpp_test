# Бэклог DSP-задач

Этот бэклог специально сделан нетривиальным и ориентированным на реальные производственные задачи обработки сигналов.

Оригинал на английском: [dsp_tasks.md](dsp_tasks.md)

## 1. Оценка TDOA с субдискретной точностью
- Цель: расширить GCC-PHAT параболической/интерполяционной аппроксимацией пика для оценки дробных задержек.
- Критерии приёмки:
  - Средняя абсолютная ошибка < 0.2 отсчёта на синтетических широкополосных сигналах.
  - Стабильная работа при SNR от 0 дБ до 20 дБ.

## 2. Адаптивный шумоподавитель (LMS + NLMS + RLS)
- Цель: реализовать модуль адаптивной фильтрации и сравнить скорость сходимости и установившуюся ошибку.
- Критерии приёмки:
  - Воспроизводимый бенчмарк-скрипт с детерминированным random seed.
  - Графики сходимости и метрики, экспортируемые в CSV.

## 3. Полифазный каналайзер
- Цель: разделить широкополосный поток на N поддиапазонов с использованием архитектуры PFB.
- Критерии приёмки:
  - Unit-тесты на идеальную реконструкцию (цепочка анализ + синтез).
  - Бенчмарк пропускной способности (Msamples/s) в режиме Release.

## 4. Блок синхронизации OFDM
- Цель: добавить грубую синхронизацию в стиле Schmidl-Cox с оценкой CFO.
- Критерии приёмки:
  - Корректное определение границ символов в многолучевом моделировании.
  - Ограниченная ошибка оценки CFO на документированном свипе SNR.

## 5. Детектор спектральных аномалий в реальном времени
- Цель: потоковый STFT + робастная оценка базового уровня (MAD или перцентили) + логика алертов.
- Критерии приёмки:
  - Метрики false positive/false negative на синтетически внедрённых событиях.
  - Задокументированный бюджет задержки в скользящем окне.

## 6. Прототип beamforming для массива микрофонов
- Цель: реализовать delay-and-sum и MVDR для небольшого микрофонного массива.
- Критерии приёмки:
  - Графики сравнения направленного усиления для нескольких углов наблюдения.
  - Метрики прироста SNR при помехах из внеосевых направлений.

## 7. Fixed-point DSP путь
- Цель: добавить Q15/Q31 варианты для выбранных ядер (FIR, Goertzel) и оценить ошибку квантизации.
- Критерии приёмки:
  - Golden-vector тесты против реализации в double.
  - Отдельные тесты на поведение при saturation/overflow.

## 8. Усиление производительности
- Цель: оптимизировать горячие участки SIMD-инструкциями (где возможно) и добавить cache-friendly блочную обработку.
- Критерии приёмки:
  - Бенчмарк-контур с сравнением baseline vs optimized.
  - CI-гейт, падающий при регрессии производительности >10%.
